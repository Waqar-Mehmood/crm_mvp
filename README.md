# The Zulfis CRM

The Zulfis CRM is a Django-based customer relationship management app for managing companies, contacts, CSV imports, and role-aware back-office workflows. It includes a branded frontend, Django admin controls, advanced filters, paginated tables, and import tooling for bringing structured contact data into the system.

## Features

- Company and contact management
- CSV import with automatic header detection and field mapping
- Role-based access control for CRM and admin access
- Branded frontend and admin experience
- Advanced filters and pagination for list views

## Tech Stack

- Django 5
- PostgreSQL 16
- Docker Compose
- Gunicorn
- WhiteNoise
- Adminer
- Shared local reverse proxy via the external Docker network `local_proxy`

## Project Structure

- `docker-compose.yml` - local app services: Postgres, Django web app, and Adminer
- `Dockerfile` - container image for the Django app
- `app/config/settings.py` - environment-driven Django settings
- `app/crm/` - CRM models, views, templates, admin customizations, auth helpers, and tests

## Local Setup

### Prerequisites

- Docker
- Docker Compose
- A machine-level local HTTPS proxy already configured to route `https://app.thezulfis.local` to the `crm-web` container on the `local_proxy` network

### 1. Create the shared Docker network

```bash
docker network create local_proxy
```

If the network already exists, Docker will report that and you can continue.

### 2. Create a local `.env`

Create a `.env` file in the repo root with these keys:

```env
DEBUG=False
SECRET_KEY=change-me
ALLOWED_HOSTS=127.0.0.1,localhost,app.thezulfis.local
CSRF_TRUSTED_ORIGINS=https://app.thezulfis.local
SITE_BRAND=The Zulfis CRM

POSTGRES_DB=crm_db
POSTGRES_USER=crm_user
POSTGRES_PASSWORD=crm_pass
POSTGRES_HOST=db
POSTGRES_PORT=5432
```

### 3. Start the services

```bash
docker compose up -d
```

### 4. Run database setup

```bash
docker compose exec -T web python manage.py migrate
docker compose exec -T web python manage.py createsuperuser
```

`collectstatic` runs automatically when the `web` container starts, so no separate first-run static build step is required.
Backend uploads are stored under `app/media/`, including `branding/` for site logos and `imports/` for uploaded CSV files.

## Local Access

This repo does not publish the app directly on a host web port. The Django app is exposed only inside Docker and is expected to be reached through a machine-level local HTTPS proxy.

Local URLs:

- `https://app.thezulfis.local/login/`
- `https://app.thezulfis.local/companies/`
- `https://app.thezulfis.local/contacts/`
- `https://app.thezulfis.local/imports/`
- `https://app.thezulfis.local/admin/`

Database UI:

- `http://localhost:8080` for Adminer

```bash
docker compose exec -T web python manage.py import_contacts_csv /app/data/contacts_import.csv --file-name contacts_import.csv
```

## Authentication and Roles

The CRM frontend uses Django users and role-based access control.

Available roles:

- `staff`
- `team_lead`
- `manager`
- `owner`

Access rules:

- CRM frontend pages require authentication
- `staff` can browse companies, contacts, and imports
- `team_lead` and above can use import upload and mapping flows
- Django admin is intended for `manager`, `owner`, and `superuser`

## Notes

- `.env` is local-only and is not committed
- `app/staticfiles/` is generated by `collectstatic`
- `app/media/` is ignored and stores backend uploads such as branding images and import CSVs
- `app/data/uploads/` is a legacy path kept ignored for older records only; new uploads go to `app/media/`
- Live deployments must mount `/app/media` to persistent storage or backend uploads will be lost on container replacement
- The shared `local_proxy` network and HTTPS proxy live outside this repo

## Live Server Deploy

The repo includes a separate server deploy path that does not depend on Docker Hub:

- `docker-compose.yml` is the local development stack and expects the external `local_proxy` network
- `docker-compose.prod.yml` is the live server stack and publishes the app directly on `8000:8000`
- `scripts/deploy_main.sh` updates the live checkout, rebuilds the app, runs migrations, and restarts the app services
- `.github/workflows/deploy-main.yml` triggers that deploy script whenever `main` is updated

### Required GitHub secrets

- `DEPLOY_HOST`
- `DEPLOY_USER`
- `DEPLOY_SSH_KEY`

### Manual server deploy

From the live server checkout:

```bash
bash scripts/deploy_main.sh
```

The default live checkout path assumed by the script is:

```bash
/home/ec2-user/crm_mvp_release
```
